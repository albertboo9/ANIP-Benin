# Plan: UI/UX Improvements for Section Titles and Card Headers

## Overview
Improve the visual hierarchy and modern design of the CahierDesCharges component by enhancing section titles and redesigning card headers to be more prominent and visually appealing.

## Current State Analysis
- **Section titles**: Currently left-aligned with icon, using `.sectionTitle` class (font-size: clamp(1.75rem, 4vw, 2.5rem), font-weight: 800)
- **Card titles**: Currently styled with `.subsectionTitle` class, includes icon, positioned below content (font-size: 1.5rem, font-weight: 700, color: primary)
- **Cards**: Use flexbox layout with icon at top, title below icon, then content

## Objectives
1. **Section Titles**: Make them uppercase, centered, slightly larger, soft and modern
2. **Card Headers**: Create dedicated header section with uppercase titles only (no icons in header)

## Implementation Plan

### Phase 1: Section Title Improvements
**Files to modify:**
- `src/components/CahierDesCharges.module.css` - Update `.sectionTitle` styles
- `src/components/CahierDesCharges.tsx` - Ensure proper structure

**Changes:**
- Convert text to uppercase using `text-transform: uppercase`
- Center align the titles using `text-align: center` and `justify-content: center`
- Increase font size slightly (from clamp(1.75rem, 4vw, 2.5rem) to clamp(2rem, 4.5vw, 2.75rem))
- Add subtle letter spacing for modern feel
- Consider removing or repositioning the decorative line (`::after` pseudo-element)
- Keep icons but make them more subtle in centered layout

### Phase 2: Card Header Redesign
**Files to modify:**
- `src/components/CahierDesCharges.module.css` - Add new `.cardHeader` class and update card layout
- `src/components/CahierDesCharges.tsx` - Restructure Card component to separate header from content

**Changes:**
- Create new `.cardHeader` class for dedicated header section
- Style header with background gradient or subtle background
- Make card titles uppercase in header
- Move icons out of header (keep them in content or remove from cards entirely)
- Adjust card layout to accommodate header-first design
- Ensure proper spacing and visual hierarchy

### Phase 3: Typography and Visual Polish
**Files to modify:**
- `src/components/CahierDesCharges.module.css` - Fine-tune typography and spacing

**Changes:**
- Adjust letter spacing for uppercase text readability
- Consider font weight adjustments for better hierarchy
- Ensure responsive behavior works well with new designs
- Test contrast and readability

## Technical Considerations
- Maintain existing animation system compatibility
- Ensure responsive design works on all screen sizes
- Keep accessibility standards (sufficient contrast, readable text)
- Consider performance impact of any new CSS properties

## Validation Steps
1. Test build process to ensure no TypeScript/CSS errors
2. Verify responsive behavior on different screen sizes
3. Check accessibility compliance
4. Ensure animations still work smoothly
5. Test visual hierarchy - titles should be clearly prominent

## Potential Challenges
- Maintaining visual balance between larger section titles and content
- Ensuring card headers don't overpower the content
- Responsive scaling of larger text sizes
- Preserving the modern, soft aesthetic while increasing prominence
