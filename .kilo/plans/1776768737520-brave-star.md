# Plan: Make ANIP-Benin Diagram Centered and Responsive

## Project Overview
This is a Next.js application for ANIP-Benin with a use case diagram that needs to be centered on the page and made responsive while maintaining proper connections between elements.

## Current State Analysis
- **Icons**: FontAwesome icons implemented ✅
- **Layout**: Fixed pixel positioning, left-aligned ❌
- **Responsiveness**: No media queries, fixed dimensions ❌
- **Connections**: Absolute pixel coordinates in SVG ❌

## Issues to Address
1. **Centering**: Diagram not horizontally centered on page
2. **Responsiveness**: Layout breaks on smaller screens
3. **Connections**: SVG paths won't scale with responsive layout
4. **Overflow**: Horizontal scrolling on smaller viewports

## Proposed Solution Architecture

### Phase 1: Container Centering
- Add flexbox centering wrapper around diagram
- Adjust `.diagram-wrapper` for horizontal centering
- Maintain vertical layout structure

### Phase 2: Responsive Grid System
- Replace absolute positioning with CSS Grid
- Define responsive grid areas for actors and modules
- Use flexible units (fr, %, vw) instead of pixels

### Phase 3: Dynamic SVG Connections
- Replace static SVG coordinates with JavaScript-calculated paths
- Implement connection lines that adapt to element positions
- Add resize event listeners for real-time updates

### Phase 4: Mobile Optimization
- Add breakpoints for tablet (768px-1200px) and mobile (<768px)
- Simplify layout for smaller screens
- Stack elements vertically on mobile if needed

## Technical Implementation Details

### CSS Grid Layout Structure
```css
.diagram-container {
  display: flex;
  justify-content: center;
  width: 100%;
  padding: 20px;
}

.canvas-area {
  display: grid;
  grid-template-columns: 80px 1fr 1fr 1fr 80px;
  grid-template-rows: repeat(12, auto);
  gap: 15px;
  max-width: 1400px;
  width: 100%;
}
```

### Element Positioning Strategy
- **Actors (left)**: `grid-column: 1; grid-row: 2/span 8;`
- **System boundary**: `grid-column: 2/span 3; grid-row: 1/span 11;`
- **Modules**: Distributed across columns 2-4 with appropriate row spans
- **Right actor**: `grid-column: 5; grid-row: 6;`

## Responsive Breakpoints
- **Desktop (>1200px)**: Full grid layout, detailed connections
- **Tablet (768px-1200px)**: Compressed grid, simplified connections
- **Mobile (<768px)**: Single column layout, minimal connections

## Success Criteria
- Diagram centered horizontally on all screen sizes
- Smooth scaling from 320px to 1920px+ viewports
- Connection lines remain accurate and visible
- No horizontal scrolling on standard devices
- Professional appearance maintained
- FontAwesome icons display correctly

## Risks & Considerations
- Complex JavaScript for dynamic connections
- Performance impact on resize events
- Browser compatibility for CSS Grid and flexbox
- Maintaining diagram readability on mobile
